Старт компонентов системы

Запуск осуществляется из командной строки. В качестве параметров при заруске передаются сетевые параметры и параметры вывода информации для клиента при необходимости, если используется второй монитор для вывода номера вызванного клиента. Для удобства создается командный файл или ярлык, который можно поместить в любое место и запускать двойным щелчком мыши. Если для установки системы использовался инсталяционный пакет, то все необходимые командные файлы и ярлыки создадутся автоматически.

После старта будет предложено выбрать от имени какого участника процесса будет выступать данный пользователь. При сворачивании окна приложения, оно сворачивается в область системных уведомлений (трей). Двойной щелчок мыши по соответствующей иконке откроет окно приложения.


       Пример командной строки запуска компонента рабочего местаоператора.


       java -cp dist/QSystem.jar ru.apertum.qsystem.server.QСlient -s 192.168.0.1 -cport 3129 -sport 3128 -cfg config/clientboard.xml


       -s 192.168.0.1 - работать с сервером системы по адресу 192.168.0.1
       -cport 3129 - порт приема сообщений клиентом 3129
       -sport 3128 - сервер работает на порту 3128
       -cfg config/clientboard.xml - файл конфигурации главного табло, не обязательный параметр. Требуется при использовании табло оператора.

 

        Пример командной строки запуска программы администрирования

         javaw -cp dist/QSystem.jar ru.apertum.qsystem.client.forms.FAdmin.


Запуск пункта регистрации

Формат команды запуска:

         javaw -cp dist/QSystem.jar ru.apertum.qsystem.client.forms.FAdmin -s 192.168.0.43 -sport 3128 -cport 31290 [debug] tach|buttons [med] [super]

  • debug - необязательный параметр, указывает на то чтобы окно не разворачивалось на весь экран и не прятался курсор мыши.
  • tach|buttons - вариант пункта регистрации, это может быть сенсорный монитор или кнопочное устройство подсоединяемое на СОМ-порт
  • med - необязательный параметр для использования пункта регистрации в мед.учреждениях для предварительной записи на прием
  • -cport 3129 - порт приема сообщений на пункте регистрации 3129
  • -sport 3128 - порт сервера 3128, с которым работает пункт регистрации
  • -s 192.168.0.1 - работать с сервером системы по адресу 192.168.0.1
  • super - необязательный параметер, указывает пункту регистрации что он должен работать с суперсайтом домена.
Запуск сервера и модуля администрирования не требует передачи параметров для запуска.
