Старт компонентов системы

Запуск компонентов

       Запуск осуществляется из командной строки. В качестве параметров передаются сетевые параметры и параметры вывода информации для клиента при необходимости. Для удобства создается командный файл или ярлык, который можно поместить в любое место и запускать двойным щелчком мыши. Для вывода сопутствующей информации в консольное окно в параметрах необходимо указать ключ debug.

Развертывание СУБД и настройка параметров соединения

Исполните инсталляционный скрипт для создания СУБД, который находится на диске. После этого создастся схема qsystem. Заведите пользователя и дайте ему права на qsystem. На сервере в jar архиве системы dist\QSystem.jar в файле ru\apertum\spring\qsContext.xml меняем параметры соединения.
Пример параметров соединения с СУБД на сервере 192.168.0.43, пользователь «admin», пароль «123».


<bean id="myDataSource" class="org.apache.commons.dbcp.BasicDataSource" destroy-method="close" lazy-init="default" autowire="default" dependency-check="default">

  <property name="driverClassName" value="com.mysql.jdbc.Driver" />

  <property name="url" value="jdbc:mysql://192.168.0.43/qsystem?characterEncoding=UTF-8" />

  <property name="username" value="admin" />

  <property name="password" value="123" />

</bean>


Запуск сервера

Запуск осуществляется из командной строки. В качестве параметра передается полное имя конфигурационного файла. Это позволяет иметь несколько конфигураций. Для удобства создается командный файл, ярлык можно поместить в любое место и запускать двойным щелчком мыши. Описание конфигурационного файла не приводится. Если используется СУБД, то никакие параметры передавать не нужно. Все необходимое для соединения с СУБД настраивается в контексте системы (описывается ниже). После запуска сервера в консольном окне будет выведено сообщение: «Сервер системы запущен» и будет производиться индикация активности сервера.


Запуск администратора

Старт можно произвести командой

         javaw -cp dist/QSystem.jar ru.apertum.qsystem.client.forms.FAdmin.


Запуск пункта регистрации

Формат команды запуска:

         javaw -cp dist/QSystem.jar ru.apertum.qsystem.client.forms.FAdmin -s 192.168.0.43 -sport 3128 -cport 31290 [debug] tach|buttons [med]

  • debug - необязательный параметр, указывает на то чтобы окно не разворачивалось на весь экран и не прятался курсор мыши.
  • tach|buttons - вариант пункта регистрации, это может быть сенсорный монитор или кнопочное устройство подсоединяемое на СОМ-порт
  • med - необязательный параметр для использования пункта регистрации в мед.учреждениях для предварительной записи на прием
